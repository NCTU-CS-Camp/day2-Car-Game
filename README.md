# day2-Car-Game

用 pygame 做的賽車小遊戲：開車繞著賽道跑，順時針繞完一圈就加分，撞到賽道邊界會重置。

## 執行方式

先同步專案依賴（第一次執行時會自動建立 `.venv`）：

```bash
uv sync
```

啟動遊戲：

```bash
uv run python main.py
```

## 操作方式

| 按鍵 | 動作 |
|---|---|
| W | 加速 |
| S | 倒車/煞車 |
| J | 左轉 |
| K | 右轉 |
| 滑鼠左鍵 | 點擊右上角 1/2/3 檔位按鈕切換最高速度 |

## 遊戲規則

- 賽道左側「START」是起點/終點線，車子在終點線附近重生。
- 賽道上有 3 個依序排列的檢查點，車子必須**順時針**依序踩過(CP1 → CP2 → CP3)、再回到終點線，才算完成一圈、分數 +1。逆時針繞圈會用錯誤的順序踩到檢查點，不會被記分。
- 撞到賽道邊界(賽道外的草地/裝飾物)會讓分數歸零，車子重生回起點。
- `Best` 會記錄這次執行期間出現過的最高分，重新執行程式就會歸零(不會存檔)。

## 檔案結構

```text
main.py                       進入點，把所有東西串在一起(讀圖、建立物件、主迴圈)
car.py                        Car 物件：油門/轉向/碰撞偵測等物理引擎(含 move()/rotation() 數學輔助函式)
race.py                       比賽規則(終點線/檢查點/分數/最高分)跟變速按鈕 UI
assets/                       車子與賽道圖片
```

## 挖空練習版(教學用)

如果想練習自己刻一遍這個遊戲的邏輯，可以參考挖空版本，裡面把判斷邏輯挖空，每個挖空都有 `Todo`/`Hint` 說明：

```bash
uv run python main_blank.py
```

填完所有空格後，玩起來應該要跟 `main.py` 一樣。題目分散在不同檔案，每個檔案裡相關的題目放在一起：

| 檔案 | 題目 | 內容 |
|---|---|---|
| `main_blank.py` | Q1 | 載入圖片 |
| `controls_blank.py` | Q2、Q3 | W/S 加速倒車、J/K 轉向 |
| `speed_and_boundary_blank.py` | Q4、Q5 | 滑鼠點擊變速按鈕、撞到賽道邊界 |
| `score_blank.py` | Q6、Q7 | 順向繞完一圈才能加分、最高分紀錄 |

`car.py` 是完整寫好的物理引擎，挖空版不會動它——不需要知道裡面怎麼算，只要知道呼叫哪個函式就會動。
